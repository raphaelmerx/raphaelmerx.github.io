How to display the Python virtualenv in your prompt using Prezto

Published Fri, Jun 10, 2016

This example uses the Prezto sorin theme. If you use a different Prezto theme, change sorin to whatever your theme name is.

Here’s what the end result looks like. I’m using Pyenv to automatically activate virtualenvs when I enter directories.

1. Add python to your plugins in ~/.zpreztorc:

# Set the Prezto modules to load (browse modules).
# The order matters.
zstyle ':prezto:load' pmodule \
  'environment' \
  'terminal' \
  'editor' \
  'history' \
  'directory' \
  'spectrum' \
  'utility' \
  'completion' \
  'git' \
  'python' \
  'prompt'

Loading the Python module gives us access to a python-info function that we’ll later use in the theme configuration.

2. Change the theme configuration to display the virtualenv in the prompt

In ~/.zprezto/modules/prompt/functions/prompt_sorin_setup, make the following changes:

1. Call python-info in the prompt_sorin_precmd function

function prompt_sorin_precmd {
  ...

  # Run python-info to get the venv name available in $python-info[virtualenv]
  python-info

  # Compute slow commands in the background.
  ...
}

Calling python-info populates the $python-info associative array with the virtualenv key.

2. In prompt_sorin_setup, define a style for the virtualenv name:

function prompt_sorin_setup {
  ...
  # Define a style for the virtualenv name
  zstyle ':prezto:module:python:info:virtualenv' format '%F{yellow}(%v)%f '

  # Set editor-info parameters.
  ...

This will display the virtualenv name in yellow, and wrapped in parenthesis.

3. At the end of prompt_sorin_setup, add the virtualenv info to the PROMPT

Replace

  PROMPT=...

with

  PROMPT='$python_info[virtualenv]${SSH_TTY:+"%F{9}%n%f%F{7}@%f%F{3}%m%f "}%F{4}${_prompt_sorin_pwd}%(!. %B%F{1}#%f%b.)${editor_info[keymap]} '

As advised in the Prezto docs, you should fork the Prezto repo, and commit your changes. Next time you switch computers, just clone your forked repository instead of the origin, and you’re set!
